# BPM Comparison Lab

キリタン壁紙の現行BPM検出と、導入候補を同じ音源で8方式同時比較する独立プレビューです。壁紙本体やCompanionの設定・データは変更しません。

## 起動

リポジトリ直下の `Run_BPM_Lab.bat` をダブルクリックします。初回だけ依存パッケージを自動インストールし、`http://127.0.0.1:5190` を開きます。

## 入力

- 音声ファイル: ファイル選択またはドラッグ＆ドロップ。ブラウザ内で再生します。
- PC音声: 画面共有ダイアログで共有対象を選び、「音声を共有」をONにします。
- マイク: Windowsで選択されている入力を解析します。
- 合成テスト: 88 / 120 / 174 BPM、および低音のない120 BPMを生成します。

## 比較方式

1. `LOW-BAND IOI`: 現行方式。低域4バケットのオンセット＋拍間隔中央値。
2. `SPECTRAL FLUX`: 全帯域の正方向変化からオンセットを検出する基準方式。
3. `SUPERFLUX-LITE`: 対数圧縮、周波数方向の最大値フィルタ、中央値＋MAD適応閾値で音程変化をオンセットと誤認しにくくした候補。
4. `AUTOCORRELATION`: 約12秒のオンセット強度履歴から繰り返し周期を推定。
5. `MULTIBAND COMB`: 低・中・高域を分け、40～240 BPM相当の周期バンクを1 BPM刻みで比較する候補。
6. `DYNAMIC PULSE`: 約12秒の履歴に拍位置列を当てはめ、拍抜けを含む大域的な一致度から推定する候補。
7. `STATE PULSE BANK`: 4つの周期・オンセット候補を状態として扱い、直前テンポからの遷移が自然な候補を優先する候補。
8. `5-WAY CONSENSUS`: 独立性の比較的高い5方式を0.5倍・2倍補正して合議する候補。

`TempoCNN`、`BeatNet`、`madmom`のRNN系は、PCM波形またはMelスペクトログラムが必要です。Wallpaper Engineから直接受け取れる128帯域値だけでは同条件比較にならないため、このプレビューでは「PCM経路が必要」と表示し、実行対象には含めていません。

曲名と既知の正解BPMを入力して「現在値を記録」を押すと、方式ごとの誤差を表へ保存できます。CSV出力はブラウザのダウンロード先へ保存されます。

## 注意

- ブラウザのPC音声キャプチャ可否は共有対象とDRMに依存します。比較を再現しやすいのは音声ファイル入力です。
- 周期方式はコールバック時刻を30 Hzの等間隔系列へ補間します。約12秒窓と5秒ロックがあるため、曲の安定した区間を15秒以上流してから記録してください。
- このラボの結果を確認してから、採用する方式だけを壁紙本体へ移植します。
