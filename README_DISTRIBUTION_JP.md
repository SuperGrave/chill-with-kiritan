# Chill with Kiritan v0.9.2 かんたん導入

この手順は、まとめ版 `Chill-with-Kiritan-v0.9.2-release.zip` を使う場合のものです。ZIPの中から直接起動せず、最初に右クリックして「すべて展開」してください。

## 1. Companionを入れる

1. 展開先の `companion/Tohoku Companion_0.9.2_x64-setup.exe` を実行します。
2. Windows SmartScreenが出た場合は、入手元を確認してから「詳細情報」→「実行」を選びます。
3. Companionを起動します。操作画面と `http://127.0.0.1:40313` のローカルAPIが一緒に立ち上がります。

インストールせず試したい場合は、同じフォルダの `tohoku-companion.exe` を直接起動できます。ウィンドウを `×` で閉じるとタスクトレイへ入り、完全終了はトレイアイコンの右クリックメニューから行います。

## 2. Wallpaper Engineへ入れる

1. `wallpaper-engine/Chill with Kiritan` フォルダを開きます。
2. Wallpaper Engineの「ファイルから開く」または「Create Wallpaper」から、まず `project.json` を選びます。
3. Web wallpaperとして保存し、デスクトップへ適用します。

`project.json` にはスペクトラム用の音声処理設定が入っています。環境によって選択できない場合だけ `index.html` を選び、Wallpaper Engine側で音声入力を許可してください。天気・ニュース・Spotify・メモ・表示設定を連動させる間はCompanionを起動しておきます。

## 3. VRMモデルを選ぶ

公開配布物には、ライセンス上VRMモデルを同梱していません。

1. Companionの `STUDIO（調律）> 3Dモデル` を開きます。
2. `VRMファイル` で、手元の `.vrm` を選択します。
3. 壁紙を一度別のものへ切り替えて戻すか、Wallpaper Engineで再読み込みします。

選択したモデルはローカルのCompanionデータフォルダへコピーされ、壁紙側で基本読み込みを試します。VRM 0.x / 1.0を想定していますが、モデルごとにボーン・表情・モーション・小物位置の互換性が異なります。

従来の既定モデル方式も使えます。Wallpaper Engineが実際に読み込んでいるフォルダへ、次の名前で配置してください。

```text
models/kiritan.vrm
```

モデルを含めたフォルダやZIPは再配布しないでください。

## 4. 画面プリセットを適用する

Companionの `STUDIO（調律）` 上部に、次の4プリセットを同梱しています。

- `1920×1080用サンプル`
- `1920x1080用サンプル(スペクトラム含)`
- `1920×1200用サンプル`
- `1920x1200用サンプル(スペクトラム含)`

使いたいプリセットの「適用」を押します。既存環境へ更新した場合も、同名の自作プリセットは残し、不足している同梱プリセットだけ追加します。

4Kでは `STUDIO（調律）` 上部の画面サイズから `3840x2160` を選び、パネル位置を調整して新しいプリセットとして保存してください。

## 5. Spotifyを使う

Spotify Developer Dashboardでアプリを作成し、Redirect URIへ次を完全一致で登録します。

```text
http://127.0.0.1:40313/spotify/callback
```

Companionの `SYSTEM（環境）> Spotify` にClient IDとClient Secretを保存し、「Spotify認証を開く」から認証します。

- `再生情報の同期間隔（秒）`: 1～60秒で調整できます。既定は2秒です。
- `曲の終了予測から0.8秒後にも更新する`: 曲切替を早く反映します。
- 通信失敗やSpotify側の制限時は、リクエスト間隔を自動的に延ばします。
- 一度取得できた歌詞は `%APPDATA%\tohoku-companion\lyrics-cache.json` に最大500曲保存され、次回からすぐ表示します。

## 6. 個人ニュースを使う

初回起動時に、青空文庫6作品、興味別オリジナル8本、日付入りニュースサンプル1本の計15候補が `%APPDATA%\tohoku-companion\personal_news_scripts` に作られます。テキストを追加・編集した後、`CONTENT（中身）> 個人ニュース` の「原稿を再読込」で反映します。既存の同名ファイルは上書きしません。

通常の本文行は、改行や句点で分断せず一続きの電光掲示板として流れます。次の表示へ切り替えたい場所だけ、独立した行へ `[Break]` と書きます。`[Paragraph]`、`[段落]`、`---` も区切りとして使え、`[Topic: 見出し]` は章見出し兼区切りになります。

歌詞がない間だけ個人ニュースを出す設定は、`REMOTE（操作）` の「歌詞なし時は個人ニュース」から切り替えられます。

## 7. スペクトラムとBPM連動を使う

1. スペクトラム対応プリセットを適用します。
2. Windowsで音楽を再生します。
3. `STUDIO（調律）> スペクトラム` の入力表示が `Companion PCM (WASAPI)` になり、BPMが確定するまで待ちます。

PCM BeatRootは、最低信頼度を超えた推定だけを表示とモーションへ渡します。解析窓、解析間隔、最低信頼度、別BPMへの切替確認時間、BPMロスト後のモーション継続時間を調整できます。曲が変わっても以前のBPMを引きずる場合は「BPM履歴を今すぐリセット」を押してください。

イヤホン、USB DAC、Bluetooth機器などへWindowsの既定出力を切り替えた場合は、自動で音声入力へ再接続します。数秒待っても戻らない場合は、Windowsの既定出力を確認してから手動リセットしてください。

「通常作業中は首だけBPM連動」を有効にすると、通常作業モードでは胸・肩・手を動かさず、頭と首だけを弱く拍へ同調させられます。

## 8. データをバックアップする

`SYSTEM（環境）> データ / バックアップ` の「書き出し」を押すと、次のデータを `%APPDATA%\tohoku-companion\backups` へJSONで保存します。

- 設定と表示プリセット
- メモとリンク
- タイマー状態
- 個人ニュースの選択・再生状態

APIキーやSpotifyの秘密情報は既定では入りません。必要な場合だけ「APIキーも含める」をONにし、そのバックアップを共有しないでください。

「読み込み」は全項目を先に検証し、1項目でも壊れていれば現在のデータを変更しません。正常な場合は現在のデータを `.bak` に退避してからまとめて復元し、Companion画面を自動再読込します。

次のファイル本体はJSONバックアップに含まれないため、必要ならフォルダごと別に保管してください。

- 背景画像・動画
- VRMモデル
- 個人ニュース原稿
- ダウンロード済み歌詞キャッシュ

## 9. 更新するとき

1. Companionを完全終了します。
2. 新しいまとめ版ZIPを別フォルダへ展開します。
3. 新しいCompanionインストーラーを実行します。通常は `%APPDATA%\tohoku-companion` の既存設定を引き継ぎます。
4. Wallpaper Engineへ新しい `project.json` を取り込み直します。
5. フォルダ配置方式のVRMを使っている場合は、新しい実フォルダにも `models/kiritan.vrm` を配置します。
6. Wallpaper Engineで壁紙を明示的に再適用または再読み込みします。

更新前にバックアップを書き出しておくと安全です。Wallpaper Engineが読み込むフォルダを直接上書きする場合は、ローカルのVRMを消さないよう注意してください。

## 困ったとき

- 壁紙が `COMPANION: OFFLINE`: Companionが起動しているか、`127.0.0.1:40313` のローカル通信が許可されているか確認します。
- きりたんが出ない: `STUDIO > 3Dモデル` の選択状態、またはWallpaper Engineが実際に読み込むフォルダの `models/kiritan.vrm` を確認します。
- 選択したVRMが出ない: 別の `.vrm` を試すか「既定モデルに戻す」を押します。
- スペクトラムが動かない: `project.json` から取り込んだか、WindowsとWallpaper Engineの音声入力が有効か確認します。
- イヤホン接続後にBPMが戻らない: Windowsの既定出力を確認し、数秒待ってから「BPM履歴を今すぐリセット」を押します。
- プリセットを戻したい: 同梱プリセットの「適用」を押します。
- 詳しい説明: `START_GUIDE_JP.md` を参照します。
