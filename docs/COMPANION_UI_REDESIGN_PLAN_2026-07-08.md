# Companion UI Redesign Plan 2026-07-08

## 目的

Companion を「裏方の設定画面」から、壁紙の情報表示部と同じ世界観で扱える操作パネルに寄せる。通常時は小さく直感的に、細かい調整時は広い別窓で落ち着いて編集できる構成にする。

## 現状の問題

- 設定項目が縦に長く、どの変更が壁紙のどの場所に効くかが見えにくい。
- 表示/音楽/天気/ニュース/3D/モーション/背景が同じ密度で並び、日常操作と制作調整が混ざっている。
- パネル位置、タイマー、ニュース、3D配置などの細部設定は小窓では手狭。
- ステータス表示は実用的だが、壁紙側のモダンな情報パネルと見た目の連続性が弱い。

## 方針

1. ホームは「状態確認と日常操作」に絞る。
   - API、Wallpaper受信、天気、ニュース、Spotify、タイマー、個人ニュース、きりたん状態をカード化。
   - 各カードには現在値、更新時刻、再取得/再生/停止などの主要操作だけを置く。

2. 設定はカテゴリごとに分ける。
   - 表示パネル
   - 背景
   - 3Dモデル/小物配置
   - カメラ
   - モーション
   - ニュース/個人ニュース
   - 天気/Spotify/起動設定

3. 詳細編集は広い別窓または拡張ビューで開く。
   - Tauri の追加 WebviewWindow を使い、1280x800 程度の「制作設定」ウィンドウを用意する。
   - 通常 Companion はコンパクトなリモコン、詳細窓は編集スタジオとして扱う。

4. 壁紙側と同じ視覚言語にする。
   - 暗色半透明パネル、細い罫線、角丸控えめ、等幅風見出し、状態色を採用。
   - 重要なトグルはスイッチ、数値はスライダー+入力、配置は軸ごとのステッパーにする。
   - 設定名だけでなく、現在の反映状態と保存状態を明示する。

## 画面構成案

- Home
  - ステータスカード
  - よく使う操作
  - 最終同期/受信状態

- Display
  - パネル表示ON/OFF
  - 歌詞/個人ニュースの自動切替
  - タイマー、メモ、ニュース、音楽の主要サイズ

- Studio
  - 3Dオブジェクト配置
  - カメラ調整
  - 背景レイヤー
  - プリセット保存/比較

- Content
  - メモ
  - リンク
  - ニュース
  - 個人ニュース原稿

- System
  - Windows起動時に起動
  - Spotify/APIキー
  - データフォルダ
  - バックアップ/リセット

## 実装段階

1. 既存画面を壊さず、Home のカードUIだけ先に刷新する。
2. 設定タブをカテゴリ単位に再編する。
3. 3D/背景/パネル位置の詳細設定を別窓に切り出す。
4. 反映済み/未保存/プリセット差分の表示を追加する。
5. 最後に全体の余白、見出し、ボタン体系を壁紙側と揃える。

## 注意

当時の `v4`、現在の `v0.8.0` では実装しない。この版は Wallpaper と Companion の受信停止、カメラ/オブジェクト初期値の0戻り、設定補修の再発防止を優先する。
