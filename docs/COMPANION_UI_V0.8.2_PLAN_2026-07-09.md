# Companion UI 刷新計画 v0.8.2 (2026-07-09)

前提ドキュメント:
- [`COMPANION_UI_REDESIGN_PLAN_2026-07-08.md`](COMPANION_UI_REDESIGN_PLAN_2026-07-08.md) … 再設計の方向性（Home/Display/Studio/Content/System、別窓スタジオ案）

この計画の到達点は、Companionを「紫の管理アプリ」から「きりたん壁紙を調律する小さな放送卓」へ寄せること。
ユーザー要望の3点を設計原則に落とす。

- ① 何を操作しているか一目で分かる（対象＝ピクトグラム＋現在値＋反映状態を常に見せる）
- ② 説明過剰にしない（文字はラベルと数値だけ。長文はツールチップへ退避、意味はアイコンで持たせる）
- ③ ボタンを大きく・サイズを統一・使いやすく（縦長固定 → 横長リサイズ可へ。コントロール寸法を規格化）

## 1. 基本方針

### 1-1. 3層に分ける（混ぜない）

操作頻度と目的で3層に分離する。旧計画の Home/Content/Studio/System を、頻度軸で束ね直す。

```
┌ REMOTE（毎日のリモコン）… 状態確認 + ワンタップ操作
├ CONTENT（中身）………… メモ・リンク・RSS・個人ニュース
├ STUDIO（調律卓）………… きりたん・背景・レイアウト・プリセット
└ SYSTEM（環境）…………… 起動・Spotify・天気・ニュース源・データ
```

### 1-2. 横長・リサイズ可へ（③）

- 既定 **1120 × 720**（横長）、最小 **960 × 600**、`resizable: true`、max撤廃。
- 左に **アイコンレール（縦ナビ）**、右に内容。横幅を活かして STUDIO を「一覧＋エディタ」の左右2ペインにできる。
- 旧計画の別窓スタジオ（1280×800 の WebviewWindow）は **やらない**。1枚のリサイズ可能ウィンドウで STUDIO を全幅表示すれば足りる。別窓は状態同期・二重管理のコストが割に合わない。将来「切り離したい」要望が出たらフェーズ4で再検討。

### 1-3. HUD語彙へ寄せる（①の世界観）

`INFO_DISPLAY_UI_STYLE_REPORT` のトークンを採用し、`App.css` の紫テーマを置換する。
- 背景は深い黒/チャコール、面はスモークガラス、罫線は白の低alpha。
- 状態色（緑/オレンジ/赤）は小面積の状態表示だけに使う。タブや主要ボタンを色分けしない。
- 大見出しは英大文字・広い字間・下罫線。数値は等幅で主役級に大きく。

### 1-4. 文字を減らす（②）

- 各コントロールは **ピクトグラム＋対象名（短語）＋現在値/状態バッジ** の3点で構成。説明文は置かない。
- 補足が要る項目は **`?`/長押しツールチップ** に退避。画面には出さない。
- 再生・章送り・カメラ移動など「動作」はアイコン単体（ラベルなし、`aria-label`とtooltipで担保）。
- 「反映済み/反映待ち/保存済み」など状態は色＋短語のバッジで表現し、文章化しない。

│  ●REMOTE │──────────────────────────────────────────────────── │
│  ○CONTENT│                                                        │
│  ○STUDIO │              コンテンツ領域（セクションごと）              │

│  ⟳ 更新   │                                                        │

└───────────────────────────────────────────────────────────────┘

- **レール**: 幅 72–84px。アイコン＋極小ラベル（現行 nav-btn の語彙を縦に）。4セクション＋下部にグローバル操作（全更新・バージョン/状態）。
- **ヘッダー**: 左にセクション名（英大文字・字間広め）、右に常時 **状態ドット列**（API/壁紙/Overlay/Spotify/天気）。どのセクションにいても壁紙との接続状態が視界に入る。→ 現行の Status タブの主要情報を常設化し、Status単独タブは STUDIO/SYSTEM の点検パネルへ縮小。

```jsonc
{
  "width": 1120, "height": 720,
  "minWidth": 960, "minHeight": 600,
  "resizable": true
  // maxWidth / maxHeight は削除
}
```

## 3. デザイントークン（`App.css` 置換）

紫アクセント（`--accent: #8c9df0` 系）を廃し、HUDトークンへ。レポートの `--kw-*` を companion 用に採用する。
**［決定②反映］モノクロだが純黒/純白の2値ではなく、不透明グレーの段階を混ぜて面と面を判別できるようにする。**
alpha重ねの半透明面だけだと背景次第でカードの境界が沈むため、レール・カード・エディタ面には**実色のグレー階調**を割り当て、コントラストで区切る。スモークガラス感は主要カードの装飾に限定して残す。

```css
:root {
  /* 面：実色グレー階調（判別性の主役） */
  --bg:            #0e0e10;   /* ウィンドウ地 */
  --panel:         #16171a;   /* レール / ヘッダー */
  --card:          #1c1e22;   /* カード・エディタ面 */
  --card-2:        #232529;   /* カード内の入れ子・トラック地 */
  --card-hover:    #2b2e34;
  --glass:         rgba(0,0,0,0.34);  /* スモークガラス装飾（主要カードのみ・任意） */
  /* 罫線：グレー＋白alphaの二段構え */
  --line:          #33363d;              /* 通常の区切り（実色で確実に見える） */
  --line-soft:     rgba(255,255,255,0.08);
  --line-strong:   #4a4e57;              /* 強調・アクティブ枠 */
  /* 文字：白〜グレーで階層 */
  --text:          #f2f3f5;
  --text-dim:      #a2a6ae;   /* メタ情報（実色グレーで可読性確保） */
  --text-faint:    #6d717a;
  /* 状態色（小面積のみ・判別のための彩度は確保） */
  --ok:   #5ecfa0;  --warn: #e6b25a;  --err: #ef7382;
  /* 選択/アクティブ：白のグロー＋明るいグレー面（色に頼らない強調） */
  --active-face:   #2f333a;
  --active-line:   #6a6f79;
  --focus-ring:    rgba(255,255,255,0.22);
  /* 形 */
  --radius:        12px;   --radius-sm: 8px;   --radius-ctl: 6px;
  --blur-panel:    blur(16px);
  --shadow-panel:  0 10px 30px rgba(0,0,0,0.45);
  /* 書体 */
  --font-head: 'WD-XLLubrifontJPN-Regular','Noto Sans JP',sans-serif; /* 見出し/数値 */
  --font-body: 'Noto Sans JP','Zen Kaku Gothic New',sans-serif;       /* 本文 */
  /* 斜線ゲージfill（進捗/スライダー用） */
  --track-fill: repeating-linear-gradient(-45deg,
    rgba(255,255,255,0.9) 0, rgba(255,255,255,0.9) 6px,
    rgba(255,255,255,0.20) 6px, rgba(255,255,255,0.20) 11px);
}
```

- **判別性の作り方**（決定②）: 地→レール→カード→入れ子 を `#0e→#16→#1c→#23` のグレー段で明確に分ける。境界は実色 `--line (#33363d)` を基本にし、白alpha罫線は補助に回す。これで暗い壁紙SS上でも面が沈まない。
- アクティブ表現は「色つきpill」ではなく **明るいグレー面（`--active-face`）＋白めの罫線＋内側1px glow**。色ではなく明度差で選択を示す。
- primary-btn のグラデ発光は撤去。primaryは「白罫線を強め、文字を白、面を一段明るいグレー」で表す。
- 状態色は判別できる最低限の彩度を保ちつつ、ドット/バッジなど小面積だけに使用（レポート指針）。
- 情報表示部と同じ見出し規約（22px級・字間0.15em・下罫線）をセクション見出しに使う。

`icons.tsx` は現在15種。ピクトグラム主体にするため、**同一スタイル（24グリッド・stroke 2・角丸端）で不足分を追加**する。色は付けない（currentColorのまま単色）。

- **禁止**: 1コントロールに2行以上の説明文。現行 `home-toggle` の `<small>note</small>`（「日時パネル」「RSS見出し」等）は撤去し、アイコンで意味を担わせる。

## 5. コンポーネント規格（③ サイズ統一）

寸法を規格化し、全画面で同じ部品を使い回す。

| 部品 | 用途 | 規格 |
|------|------|------|
| **アクションボタン** | 主要動作 | 高さ 44px・最小幅 96px・アイコン18＋短語・角丸 `--radius-sm` |
| **アイコンボタン** | 動作単体 | 44 × 44px（現行の6px paddingから拡大）・tooltip |
| **トグルタイル** | パネルON/OFF等 | 固定 高さ 64px、`grid: アイコン48 / 対象名 / ONOFFバッジ`。8枚が横に並ぶ自動グリッド |
| **スイッチ** | 真偽値 | iOS型トグル（現行checkboxを置換）幅 46 × 高 26px |
| **スライダー行** | 数値 | 全幅レール（斜線fill）＋右端 72px 数値入力。1行=`ラベル / スライダー / 数値` の固定3分割 |
| **ステッパー** | 位置/回転など軸値 | `− [値] +` 一体型、高さ 40px |
| **方向パッド** | カメラ/移動 | 3×3グリッド・各セル 48 × 44px（現行42/36から拡大） |
| **セグメント** | モード選択(2–4択) | 等幅セグメントボタン。selectを置換して指で押しやすく |
| **状態バッジ(pill)** | 状態表示 | 現行pill流用、色は状態色のみ |
| **カード** | リスト項目 | `--surface-2`＋`--line`。ページ全体をカード化しない（レポート指針） |

- タップ領域は最小 40px を全部品で厳守（現行の 36px・11px小ボタンを底上げ）。
- `control-grid` は幅に応じて 2→3→4 列へ。固定2列をやめる。

## 6. 画面別 詳細ワイヤー

### 6-1. REMOTE（毎日のリモコン）

目的: 起動して最初に見る画面。状態が読めて、よく使う操作が指1本。

```
REMOTE ─────────────────────────  ●API ●壁紙 ●天気 ●Spotify
┌ パネル表示 ─────────────────────────────────────────────┐
│ [🕐時計 ON] [☁天気 ON] [📰ﾆｭｰｽON] [♪音楽 ON]              │
│ [🎤歌詞 OFF][📡個人 ON][📝ﾒﾓ ON] [⏲ﾀｲﾏｰON]              │  ← 64pxトグルタイル×8
└──────────────────────────────────────────────────────┘
┌ タイマー ──────────────────┐ ┌ いま更新 ───────────────┐
│ FOCUS / SET 02              │ │ [☁天気] [📰ﾆｭｰｽ] [⟳全部] │
│      25:00                  │ └────────────────────────┘
│ [▶] [⟳] [⏭]                │
└────────────────────────────┘
```

- パネルトグルは説明文なし。アイコンで対象、右肩に ON/OFF バッジ。
- タイマーは数値主役（`--font-head`、大）。操作はアイコン。
- 「いま更新」= Status タブにあった手動更新を集約。状態ドットはヘッダー常設なので Status タブは廃止級に縮む。

### 6-2. CONTENT（中身）

サブセグメントで メモ / リンク / RSS / 個人ニュース を切替（横長なので上部セグメント）。

- **メモ**: 追加欄＋カード一覧。ピン・削除はアイコン（ホバーで出す現行踏襲だが常時表示化を検討）。
- **リンク**: タイトル/URL 追加＋カード。頭文字アバターは踏襲。
- **RSS**: 左=フィード一覧（状態バッジ）、右=選択フィードの記事。横幅を活かし2ペイン。
- **個人ニュース**: 上=「再生中カード」（章/進捗/現在行）、中央=大きな再生トランスポート（アイコン列 ⏮ ◀ ▶ ▶▶ ■）、下=章リスト＋補足リスト。再生系は全アイコン。

### 6-3. STUDIO（調律卓）— 最重要

現行 TabDisplay の全機能を、左「対象一覧」＋右「エディタ」の2ペインへ再編。縦積みアコーディオンを廃止。

```
STUDIO ─────────────────────────────────  画面 [1920x1200 ▾]  反映●済
┌ 対象 ───────┐ ┌ エディタ: きりたん ───────────────────────┐
│▸ きりたん    │ │ [表示 ◉]  明るさ ▊▊▊▁▁ 1.0                  │
│  背景        │ │ 小物  [📱ON][🎮ON][☕OFF]                   │
│  レイアウト   │ │ ─ 配置 ─                                    │
│  カメラ      │ │ 位置 X −[0.00]+  Y −[..]+  Z −[..]+          │
│  モーション   │ │ 回転 X −[..]+   …                            │
│  ─────       │ │ 拡大 …                          [初期値に戻す] │
│  プリセット   │ │ ─ 小アクション/モード除外 …（チップ選択）        │
└─────────────┘ └───────────────────────────────────────────┘
```

STUDIO のサブ対象:

1. **きりたん**: 3D表示/明るさ/VRM・小物ON/OFF・3D配置(位置/回転/拡大)・モーション(自動/固定・間隔・除外)。
   - ⚠ VRM未読込時はカメラ/配置が効かない既知挙動 → **「VRM未読込」バッジ**を出し、無効な操作をグレーアウト（`wallpaper-companion-pna-cors` の混乱を予防）。
2. **背景**: 画像/動画/キュー/オーバーレイ。ここは**サムネイル・グリッド**にして「今何が入っているか」を視覚化。追加は大きなドロップ領域。オーバーレイは行カードで濃度/合成/フィット。
   - 注意: 背景メディアは既存どおり backgrounds フォルダ保存＋URL参照を厳守（`companion-ui-payload-media-bloat`）。プレビューに base64 を settings へ戻さない。

4. **カメラ**: 調整ON/OFF・ideal差分・移動/回転パッド（大型化）・ステップ・0戻し。
5. **モーション**: 自動/固定・固定モード・間隔プリセット・除外モード/小アクション（チップ）。
6. **プリセット**: 保存/適用/上書き/名前/削除。アクティブ明示。将来「差分表示」（旧計画4）はここに追加余地。

- 上部に **画面サイズ選択** と **反映状態バッジ**（自動反映中/反映待ち/反映済み）を常設。現行の自動保存（180msデバウンス→putUi）ロジックはそのまま活かす。

### 6-4. SYSTEM（環境）

グループカードを縦に。各グループ見出しはアイコン＋短語。

- **起動**: ログイン時起動スイッチ／最高権限スイッチ／登録状態バッジ／［状態確認］［管理者権限で登録］。長い注意書きはツールチップへ。
- **Spotify**: Client ID／Secret（write-only）／Refresh Token＋設定済みバッジ、［認証を開く］［接続確認］。Redirect URI はコピー可能な小コード表示。
- **天気・地域**: 都道府県セグメント/セレクト・表示名・緯度経度・JMA・［保存して更新］。
- **ニュース源**: フィードURL（複数行）・最大件数・［保存して更新］。
- **データ / バックアップ**（新規・v0.8.2で実装／§6-5）: 設定の書き出し・読み込み、データフォルダを開く。

### 6-5. 設定バックアップ（決定④で実装確定）

Companionのデータは `%APPDATA%\tohoku-companion` に平易なJSONで保存されている（[`state.rs`](../03_companion/src-tauri/src/state.rs)）。
`companion-data.json` に **UIプリセット・全表示/3D/カメラ/背景/モーション設定・メモ・リンク・タイマー・アプリ設定（天気/ニュース/起動）** が入り、機微な `secrets.json`（APIキー）は別ファイル。
既に原子的書き込み＋`.bak`退避（`write_atomic_with_backup`）が整っているため、書き出し/読み込みの追加は小さく安全。

UI（SYSTEM > データ）:

```
データ / バックアップ ────────────────────
[⤓ 書き出し]   [⤒ 読み込み]   [🗀 フォルダを開く]
□ APIキーも含める（既定OFF・機微情報の注意）
最終バックアップ: 2026-07-09 14:20  •  復元後は壁紙が次回同期で反映
```

- **書き出し**: 保存ダイアログ → `kiritan-companion-backup-YYYYMMDD.json` を書き出し。既定は `companion-data.json` 相当（secrets除外）。opt-in時のみ `secrets` を同梱（別セクションに隔離してエクスポート）。
- **読み込み**: ファイル選択 → 検証（スキーマ/バージョン）→ 現状を自動 `.bak` 退避してから適用 → `persist()` → 壁紙は次回ポーリングで反映。失敗時はロールバック。
- **フォルダを開く**: tauri opener で `data_dir` を開く（軽い先行実装）。

必要な backend 追加（最小）:

| 追加 | 役割 |
|------|------|
| `GET  /api/backup/export?includeSecrets=bool` | 現在のデータをバックアップJSONで返す（既定secrets除外） |
| `POST /api/backup/import` | バックアップJSONを検証し、`.bak`退避のうえ state へ適用・`persist()` |
| `GET  /api/data-dir`（or tauri command） | データフォルダのパス返却（「フォルダを開く」用） |

フロント `api.ts` に `exportBackup()/importBackup()/dataDir()` を追加。既存の `getUi/putUi`・`getSettings/putSettings` と同じ経路。
リセット（初期化）は誤操作リスクが高いので v0.8.2 では出さない（フォルダを開いてファイル削除で代替可能）。要望が強ければ後続で「二段確認つき初期化」を追加。

## 7. テキスト削減の具体（②の徹底）

現行で画面に出ている説明文の扱い:

| 現行の文言 | 新扱い |
|-----------|--------|
| home-toggle の note（「日時パネル」等） | 撤去（アイコンで代替） |
| 「スライダーを動かすと…取り込みます」 | 反映バッジのtooltipへ |
| 起動グループの長い hint | `InfoIcon` tooltip |
| 「0がidealカメラそのものです…」 | カメラ見出し脇のtooltip |
| Spotify Redirect URI 手順 | コピー欄＋tooltip |
| 個人ニュース「本文は連続スクロール中です」等 | 現在行カードの淡色補足のみ |

原則: **画面に残す文字＝ラベル・数値・状態語だけ**。手順/理由/背景は tooltip。

## 10. リスク・留意

- **背景メディア肥大**（`companion-ui-payload-media-bloat`）: サムネイル/プレビュー実装で base64 を settings に戻さない。URL参照を維持。
- **カメラ/配置のVRM依存**（`wallpaper-companion-pna-cors`）: VRM未読込では効かない。STUDIOで状態バッジ＋無効化して誤操作感を消す。
- **自動反映の整合**: STUDIO再編でも 180msデバウンス→`putUi`→壁紙ポーリングの流れを崩さない。反映バッジで可視化。
- **情報過多の再発**: STUDIOは項目が多い。左一覧で「対象を1つ選ぶ」制約を守り、1画面に全部出さない。
- **フォント**: `WD-XLLubrifontJPN-Regular` はCompanionにも同梱/参照が要る（`02_ui-overlay/src/styles/fonts.css` 参照）。長い日本語本文はfallbackを混ぜる。
```
