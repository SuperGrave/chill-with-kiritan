# 個人ニュースパネル 実装計画（2026-07-06）

## 1. 目的

Spotifyで再生中の曲に歌詞データがない場合の受け皿として使える、独立した「個人ニュース」パネルを追加する。ニュースはユーザーの興味プロファイルに合わせて作った原稿を、きりたん寄りの短い口調で1行ずつ横スクロール表示する。

今回の対象は「表示・再生制御・原稿読み込み」の土台まで。ニュース収集と原稿生成の完全自動化は次段階とし、まずはCompanionが指定フォルダの原稿を読み込み、壁紙側にライブ状態として配信できる状態を作る。

## 2. 画面仕様

個人ニュースは歌詞パネルとは別の独立パネルとして表示する。通常は歌詞パネルと同時表示でき、設定によって「歌詞行が0件の時だけ個人ニュースを自動表示し、その間は歌詞パネルを隠す」動作にもできる。

パネルは3段構成を基本にする。

1. 上段: `PERSONAL NEWS : 原稿タイトル` をパネルヘッダー側に表示し、本文内にも原稿タイトルと状態バッジを表示できる。
2. 中段: 現在のトピック/サブタイトルを表示し、その下で本文を1行ずつ横スクロールする。
3. 下段: 全体進捗、現在行、総時間を表示し、再生バー上にチャプターと参考リンクのマークを置く。

表示要素は他パネルと同じ思想で、表示/非表示とサイズ調整をCompanion/Overlay設定から行う。

- 通常表示
- 歌詞なし時の自動表示
- 自動表示中に歌詞パネルを隠すか
- タイトル表示
- トピック表示
- 本文表示
- 参考リンク表示
- 進捗バー表示
- チャプター/リンクマーク表示
- タイトル/トピック/本文/参考リンクの文字サイズ
- 行間
- 進捗バー高さ
- 横スクロール速度

## 3. 原稿フォーマット

ずんきりラジオの初期原稿フォーマットを参考に、Markdown風のメタ情報と `## Scenario` 以降の進行命令で構成する。キャラクター名・話者指定は今回入れない。

配置先はCompanionが読む `personal_news_scripts` フォルダ。開発用には `03_companion/personal_news_scripts/` を用意し、実行時データディレクトリ側の `personal_news_scripts` も探索対象にする。

```text
# Title: 2026-07-05 俺流興味ニュース
# Description: 歌詞が見つからない曲の時に流す5分くらいのニュース原稿。
# DefaultLineMs: 9500

## Scenario
[Topic: 01 AIコーディング安全メモ]
[Line: 8.5] きりたんです。今日は昨日の俺流興味ニュースをざっくり流します。
[Source: TechRadar - Claude Code exploit report | https://example.com/article | 5.0]
[Line: 10.0] このニュースは、AIが親切に直そうとして実行する瞬間が攻撃面になる、という話です。
(Wait: 1.5)
```

命令の意味:

- `# Title:`: ファイル名とは別の表示タイトル。
- `# Description:`: Companion側の説明文。
- `# DefaultLineMs:`: 秒指定のない本文行の既定表示時間。ミリ秒。
- `[Topic: ...]`: サブタイトル/チャプター。番号は原稿側で書く。
- `[Line: seconds] text`: 本文1行。指定秒数で横スクロールと進捗を進める。
- `[Source: title | url | seconds]`: 参考リンク。本文には `参考: title` と出し、リンク一覧とバー上マークにも使う。
- `(Wait: seconds)`: 間を置くための待機行。
- 通常テキスト行: 本文行として扱う。1物理行に複数文がある場合は、読み込み時に日本語の句点・感嘆符・疑問符で1表示ずつへ正規化する（元ファイルは変更しない）。

## 4. Companion操作

Companionには「ニュース」タブを追加する。

できる操作:

- 原稿一覧の再読込
- 原稿選択
- 再生/一時停止
- 停止
- 最初から再生
- 前/次チャプター
- 任意チャプターへジャンプ
- リピートON/OFF
- 原稿内の参考リンク一覧表示
- 参考リンクを1タッチでブラウザに開く

APIは以下を追加する。

- `GET /api/personal-news`
- `POST /api/personal-news/reload`
- `POST /api/personal-news/select`
- `POST /api/personal-news/control`

`GET /api/runtime` には `personalNews` を追加し、Overlay/Wallpaperが通常のポーリングで取得できるようにする。

## 7. 青空文庫取り込み

青空文庫のルビ付きzipを個人ニュース原稿形式へ変換する補助ツールを用意した。

```powershell
pwsh -NoProfile -ExecutionPolicy Bypass -File .\tools\import_aozora_personal_news.ps1
```

変換処理は、zip取得、Shift-JISテキスト展開、青空文庫のルビ/入力注記の軽い除去、文単位の `[Line: seconds]` 化、図書カードURLの `[Source: ...]` 挿入までを行う。出力先は `03_companion/personal_news_scripts/`。

初期サンプルとして以下を生成済み。

- `aozora_夢十夜_第一夜.txt`
- `aozora_羅生門_冒頭.txt`
- `aozora_走れメロス_冒頭.txt`
- `aozora_銀河鉄道の夜_冒頭.txt`
- `aozora_注文の多い料理店_冒頭.txt`
- `aozora_檸檬_冒頭.txt`

## 8. 受け入れ条件

- 歌詞がある曲では従来の歌詞表示が崩れない。
- 個人ニュースは歌詞とは別パネルとして単独表示できる。
- 歌詞パネルと個人ニュースパネルを同時表示できる。
- 歌詞がない曲では、設定により歌詞パネルを隠して個人ニュースパネルを表示できる。
- 原稿タイトル、現在トピック、本文、進捗バーが表示される。
- Companionから再生/停止/チャプター移動/リピート/原稿選択ができる。
- 参考リンクが本文内のsource行、バー上マーク、Companionリンク一覧に反映される。
- 青空文庫サンプル原稿がCompanionの原稿一覧に出て、個人ニュースパネルへ流せる。
- 01/02/03のproduction buildとRust testが通る。
