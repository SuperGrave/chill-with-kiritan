# Chill with Kiritan v0.8.2

Generated: 2026-07-10

## Changes

### Companion UI 刷新（v0.8.2 の主目的）

- ウィンドウを 440x700 固定縦長から 1120x720（最小 960x600）のリサイズ可能な横長へ変更。
- 紫アクセントの管理アプリ調テーマを廃止し、壁紙情報表示部と同じモノクロHUD語彙へ全面移行。
  純黒/純白ではなく実色グレー階調（地 #0e0e10 → カード #1c1e22 → 入れ子 #232529）と
  実色罫線で面を判別する。状態色（ok/warn/err）はドット・バッジの小面積のみ。
- 見出し・計器数字に壁紙と同じ `WD-XLLubrifontJPN-Regular` を同梱・適用。
- 下部タブ → 左アイコンレール（操作 / 中身 / 調律 / 環境）＋ヘッダー常設の接続状態ドット
  （API / 壁紙 / 天気 / Spotify）。Status 単独タブは廃止し REMOTE の「いま更新」へ吸収。
- REMOTE: パネル表示 8 トグル（64px タイル・アイコン＋状態バッジ・説明文なし）、
  等幅大数字タイマー、手動更新の集約。
- CONTENT: メモ / リンク / RSS / 個人ニュースをセグメント切替に集約。
  RSS は左フィード＋右記事の 2 ペイン。個人ニュースは大型ピクトグラムの
  再生トランスポート（最初から / 前章 / 再生・一時停止 / 次章 / 停止 / リピート）。
- STUDIO: 旧・表示設定の縦積みアコーディオンを解体し、
  対象一覧（きりたん / 背景 / レイアウト / カメラ / モーション / システム）＋
  エディタの 2 ペインへ再構成。自動反映・プリセット・D-pad 等の全機能は維持。
  - きりたん / カメラのペインに VRM 読込済・未報告バッジを追加
    （壁紙の kiritan 状態報告の鮮度から判定。未報告時はツールチップで
    models/kiritan.vrm の確認を案内）。
  - 背景キューにサムネイルグリッドを追加（画像プレビュー / 動画タイル /
    個別削除。設定には従来どおり URL のみ保持し base64 は持たない）。
  - レイアウトにキャンバス比率の簡易プレビューを追加（各パネル矩形が
    ドラフト値でライブ更新。非表示パネルは破線表示）。
- 共通コントロール規格 `controls.tsx`（Button 44px / IconButton 44x44 /
  Switch / ToggleTile 64px / Segment / SliderRow / Stepper / Pill / Card / InfoHint）。
- 長い説明文は見出し脇の ⓘ ツールチップ（InfoHint）へ退避。

### 設定バックアップ（新機能）

- `POST /api/backup/export` … 設定・UIプリセット・メモ・リンク・タイマーを
  `%APPDATA%\tohoku-companion\backups\` へ書き出し。APIキー（secrets）は既定で除外、
  「APIキーも含める」を有効にした時だけ同梱。
- `POST /api/backup/import` … 現データを `.bak` へ退避してから適用・永続化。
  起動登録も再整合。
- `GET /api/data-dir` … データフォルダの場所を返す（「フォルダを開く」用）。
- SYSTEM > データ / バックアップ に 書き出し / 読み込み / フォルダを開く を追加。

### 個人ニュースの修正

- 読み上げ行の表示時間を一律 10 秒から文字数比例（180ms/文字・下限 2 秒、
  インライン補足マーカーと URL は文字数に数えない）へ変更。
  `[Line: Xs]` の明示指定と `# DefaultLineMs:`（Wait 行の既定）は従来どおり尊重。
  これにより overlay のティッカー速度が原稿によらず約 5.6 文字/秒で一定になり、
  短文原稿で異常に遅く・長文で速すぎる問題と、チャプター/補足タイミングの
  ズレを解消。
- 同梱の青空文庫サンプル 4 本から、旧式で誤較正されていた `[Line:]` 値を除去し
  新しい自動見積りに統一。
- 原稿検索ディレクトリが重なった場合に同一原稿を二重登録して ID が衝突する
  バグを修正（React の duplicate key 警告の原因）。
