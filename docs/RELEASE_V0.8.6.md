# v0.8.6 Release Notes

v0.8.6は、v0.8.5のSpotify・4K・VRM対応へ、歌詞キャッシュと個人ニュースの追加修正をまとめた正式リリースです。

## Changes since v0.8.5

### 歌詞キャッシュ

- LRCLIBから取得できた歌詞を `%APPDATA%\tohoku-companion\lyrics-cache.json` に保存。
- 過去に再生した曲はネットワークへ再問い合わせする前にローカルキャッシュを利用。
- キャッシュは最大500曲とし、空データや取得エラーは保存しない。
- 壊れたキャッシュは安全に無視し、次回取得時に置き換える。

### 個人ニュース

- 初回候補を15本へ拡充。青空文庫6本、興味別オリジナル8本、日付入りニュース1本を同梱。
- 日本語ファイル名から作る原稿IDの衝突を解消し、旧IDから選択中原稿と再生位置を引き継ぐ互換処理を追加。
- 1物理行に複数の文がある原稿を、読み込み時に `。` `！` `？` 単位の1文1表示へ正規化。元ファイルは変更しない。
- Overlayの本文を原稿全文の単一マーキーから、backendの現在LINEと同期する1行表示へ変更。
- Spotify歌詞が再取得された際にもニュースの時間軸を停止せず、続きを維持。
- Companionへ現在の `LINE` 番号と前後の行へ移動する操作を追加。
- 青空文庫取り込みツールも文単位の原稿生成へ対応。

## Included from v0.8.5

- Spotify同期間隔の設定、通信失敗時のバックオフ、曲終了約0.8秒後の追加更新。
- 4K `3840x2160` 表示設定。
- 任意VRMファイルの選択と読み込み試行。
- 個人ニュースのバックグラウンド進行、再生位置保存、再起動後の復旧。

## Notes

- 公開パッケージにはVRM/VRMA、Spotify認証情報、APIキー、個人設定、取得済み歌詞キャッシュを含みません。
- VRM 0.x / 1.0の基本読み込みを試しますが、モデル固有のボーン、表情、モーション、小物配置は完全互換を保証しません。
