# v0.9.2 Release Notes

v0.9.2は、実用環境のBPM判定をPCM BeatRootへ一本化し、曲中の一時的な推定ブレと曲切替後の古い履歴を抑えながら、きりたんの拍同期を自然に見せる更新です。

## PCM BeatRootによるBPM推定

- CompanionがWindowsの既定再生デバイスをWASAPIループバックで取り込み、11,025 Hz・モノラルPCMを壁紙へ渡します。
- イヤホン・USB DAC・Bluetooth機器の接続などでWindowsの既定出力が変わった場合、500ミリ秒間隔で変更を検知し、新しいデバイスへ自動再接続します。
- 再接続中はBPM画面へ状態を表示し、古いデバイスの解析履歴を一度だけ初期化してから新しい音声を解析します。
- BeatRoot解析はWorkerで実行し、描画・設定画面の応答を妨げない構成です。
- 解析信頼度が70%未満の推定はBPM表示とモーションへ送りません。
- 解析窓、解析間隔、最低信頼度、別BPMへの切替確認時間をCompanionのスペクトラム設定から変更できます。

## 履歴安定化と解析リセット

- リセットされるまで確定BPMの履歴を保持し、±3 BPMの同一テンポ帯と半テンポ・倍テンポを同じ系統として集計します。
- 一時的に別の候補が現れても、既定9秒・3回以上の連続確認を満たすまでは現在の確定値を維持します。
- Spotifyの曲切替確認時、指定した分間隔、Companionの「BPM解析を今すぐリセット」から履歴と解析窓を消去できます。
- Spotify曲切替は既存の「曲終了予測+0.8秒」追加更新と連動します。

## BPMモーション

- 40〜240 BPMを1 BPM刻みで用意した固定速度のモーションバンクを使用します。
- 高速曲でもテンポを自動的に半分へ落とさず、検出BPMどおりに1拍1周期で動きます。
- BPMロスト後は設定秒数だけ同じバンク・位相を継続し、同じBPM帯へ戻れば停止せずそのまま再生します。
- 指トントンの打鍵と首振りの下端を拍の頭へ揃えました。
- 「通常作業中は首だけBPM連動」を追加しました。通常作業モードでは胸・肩・手へBPMオフセットを加えず、頭と首だけを弱く連動させます。専用の強さも調整できます。

## スペクトラム対応プリセット

- 従来の1920×1200・1920×1080サンプルに加え、ユーザー調整済みの「スペクトラム含」版を各解像度へ追加しました。
- クリーン環境と既存環境のどちらも合計4プリセットになります。同名のユーザー作成版が既にある場合は重複追加しません。
- ローカルVRMのURLは同梱せず、BPM方式は現行のPCM BeatRootへ正規化しています。

## データバックアップ

- 設定・プリセット・メモ・リンク・タイマー・個人ニュース再生状態の書き出し／読み込み往復テストを追加しました。
- 通常バックアップはSpotify秘密情報を含まず、「APIキーも含める」を明示した場合だけ別セクションへ格納します。
- 読み込み前に全項目を検証し、1項目でも壊れていれば何も変更しない方式へ変更しました。
- 読み込み成功後はCompanion画面を自動再読込し、復元した設定を即座に反映します。
- 背景・VRM・個人ニュース原稿・歌詞キャッシュのファイル本体はJSONバックアップの対象外です。

## 検証

- BPM推定・モーション結合テストで、1 BPMバンク、位相維持、高速テンポ、ロスト猶予、首と指の下端一致、通常作業の頭・首限定動作を確認。
- PCM BeatRootのWorker、信頼度ゲート、履歴安定化、手動・Spotify・定期リセットの単体テストを実施。
- Wallpaper、Overlay、CompanionのTypeScriptビルド、Rustテスト、Tauri本番ビルドを実施。
