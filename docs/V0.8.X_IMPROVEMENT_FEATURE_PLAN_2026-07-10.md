## 2. A系 — 直した方がいい部分

規模の目安: **S** = 〜半日 / **M** = 1〜2日 / **L** = 3日〜

### A1. 壁紙のCompanion同期を「変化があった時だけ反映」にする 【S・効果大】

- **現状**: `01_wallpaper/src/App.tsx` の1秒ポーリングが、応答が前回と同一でも毎回 `setWallpaperSettings` / `setMotionSettings` に新オブジェクトを渡す → App全体（埋め込みオーバーレイ含む）が**毎秒フル再レンダー**。24時間動く壁紙としてはCPUの無駄。
- **やること**: オーバーレイ側（`02_ui-overlay/src/App.tsx` の `lastSignature` 方式）と同じ署名比較を導入し、無変化ならsetStateしない。あわせて同一ページ内で二重になっている `/api/ui` ポーリング（壁紙1s＋埋め込みオーバーレイ700ms）を1本に共有化。
- **効果**: アイドル時のCPU/電力削減。体感の引っかかり予防。

### A2. Wallpaper Engine連携の仕上げ（一時停止・FPSプロパティ） 【M】

- **現状**: `project.json` のpropertiesが空。`wallpaperPropertyListener` / `setPaused` / `visibilitychange` への対応がソースに一切ない。全画面アプリ使用中もWE側の設定次第でレンダリングが回り続ける。
- **やること**:
  1. `window.wallpaperPropertyListener.setPaused` → RAF停止・背景動画pause・ポーリング間隔延長
  2. WEプロパティ（FPS上限、昼夜override等の最小セット）を `project.json` に定義し `applyUserProperties` で反映
  3. ブラウザ実行時は `document.visibilitychange` で同等の省電力化
- **効果**: ゲーム中のGPU食い防止。WEユーザー体験の標準化。

### A3. lint負債の一掃（01: 9件 / 02: 60件） 【M】

- **現状**: 02は `any` 型が大半（companionClient / SettingsPanel / weatherService）＋ `set-state-in-effect` 数件。01は `no-useless-assignment` 等6 errors。監査時点（01:5件/02:44件）から**増加傾向**で、放置すると新規エラーが埋もれる。
- **やること**: 型付け（`CompanionUi` の `Record<string, any>` を実際の設定型に）、effect内setStateの整理。lintをCIゲート化（`npm run lint` を package_release.ps1 の検証に追加）。
- **効果**: 退行検知が効くようになる。型が付くとCompanion⇔overlay間の設定スキーマ齟齬も見つけやすくなる。

### A4. cargo fmt の初適用＋ゲート化 【S】

- **現状**: `03_companion/src-tauri` はfmt差分が全域に残る（監査で既知）。
- **やること**: `cargo fmt` 一括適用（機能差分ゼロの単独コミット）→ 以後 `cargo fmt --check` を検証手順に追加。
- **効果**: 以後のRust diffが読みやすくなる。

### A5. 死にコードの棚卸し 【S】

- **現状**: `TabStatus.tsx` が未参照。`TodoItem` / AIチャットのモデル・サービスは休眠（B5/B7で活かす選択肢あり）。
- **やること**: TabStatus削除。Todo/Chatは「B系で採用」or「削除」をこの選定で決める（採用しないなら消してスリム化）。
- **効果**: 迷子コードの削減。※B5/B7を選ぶ場合、対応する削除は行わない。

### A6. テスト用データディレクトリのenv override 【S】

- **現状**: `AppState::load()` は常に `%APPDATA%\tohoku-companion` を使う。実機E2E検証のたびに実データへ書き込みが走る（監査SHOULD項目、未対応のまま）。
- **やること**: `TOHOKU_COMPANION_DATA_DIR` 環境変数（または `--data-dir` 引数）で `load_from` に流す。ついでに990KBの古い肥大`.bak` 2本を `_archive/` へ退避。
- **効果**: 以後の実機テストが安全になる。

### A7. バージョン表記の統一 【S（微修正）】

- **現状**: 03は npm/tauri/Cargo とも 0.8.2 で整合済みだが、01/02 の package.json が `0.0.0` のまま。
- **やること**: 01/02 も本流バージョンに合わせ、`package_release.ps1 -Version` から自動で書き換える。
- **効果**: 生成物の由来がバージョンで追える。

### A8. アップロードbody上限をルート限定にする 【S】

- **現状**: `api.rs` の `DefaultBodyLimit::max(512MB)` が**全ルート**に掛かっている（背景動画アップロードのための値）。
- **やること**: 512MBは `/api/backgrounds/upload` だけに `route_layer` で付け、他は既定（〜2MB）へ戻す。
- **効果**: ローカルAPIとはいえ、異常リクエストでのメモリ浪費を防ぐ衛生改善。

### A9. 背景アート3枚の導入（完成監査MUST#6の残り） 【L・素材制作/調達】

- **現状**: `public/scenes/room_workdesk_day/` にJSONのみ。夜昼の見た目差はCSSグラデーションだけ。
- **やること**: room_back / outside / light_overlay の昼夜ぶんを自作 or CC0調達 → `scene.json` に配線（スキーマ・ローダは実装済みなので**画像を置いて指すだけ**）。
- **効果**: 見た目の完成度が一段上がる。※素材の方向性（写真調/イラスト調）は要決定。

### A10. 昼夜ライティングの実機目視QA 【S・検証タスク】

- **現状**: Three.js側の昼夜光量切替はコード検証のみで、目視確認が未実施のまま（screenshot経路の制約による積み残し）。
- **やること**: `?lab=1` のfreeze+capture経路で昼夜それぞれをキャプチャして比較記録を残す。ズレがあれば光量値を調整。

### A11. kiritan state報告の充実＋Companion側表示 【S〜M】

- **現状**: 報告は mode と時刻のみ実質有効（`VrmViewer.tsx` の `maybePost` は `ambient: null, away: null` 固定）。Companion側の表示もSTUDIOのVRMバッジ（鮮度判定）どまり。
- **やること**: 進行中アンビエント・離席状態を実値で送る → REMOTEの「いま更新」付近に「きりたんは今: うとうと作業中（3分前から）」のような一行表示を追加。
- **効果**: B3/B4（連動系）の土台にもなる可視化。

## 3. B系 — 追加すると楽しそうな機能

### B1. うとうと作業モード解禁（著作済み素材の配線だけ） 【S・即効性No.1】

- **土台**: `loop_work_sleepy.motion.json` と `amb_slpy_head_bob / slow_blink / tilt_drift` は**著作済みで置いてあるのに未使用**。遷移用の `tr_sit_to_slump` / `tr_slump_wake` も著作済み。
- **やること**: `VrmViewer.tsx` の `DIRECTOR_PLAYABLE_MODES` / `DIRECTOR_LOOPS` / `DIRECTOR_AMBIENTS` に work_sleepy を追加し、遷移チェーン（work→sleepy→sleep_desk）を実機確認。CompanionのSTUDIO>モーションの固定モード選択肢にも追加。
- **楽しさ**: 「仕事してたのに、だんだん船を漕ぎ出して、机で寝落ちする」流れが完成する。深夜バイアス（設計表で lateNight ×3.0）も既にあるので、**夜更かししてる時ほど一緒に眠くなる**。

### B2. 新モード追加パック（読書 / スマホ / ゲーム / おやつ / 音楽鑑賞） 【L・本命コンテンツ】

- **土台**: `modeTable.ts` に11モードぶんの設計（遷移確率・滞在時間・アンビエント池）が完備。机上小物（book / phone / controller / snack_plate / headphones / cup）は**モデル・配置・持ち替え機構（microEvents/grip）まで実装済み**。足りないのはループ＋アンビエントのDSL著作のみ。
- **やること**: 1モードにつき loop 1本＋ambient 2〜4本＋出入り遷移を著作（MOTION_AUTHORING_GUIDEの著作ループ）。おすすめ順: ①read_book（座位のまま・小物1個で低リスク）→ ②phone_browse → ③snack_break → ④game_controller → ⑤music_listen。
- **楽しさ**: 一日の生活が「仕事・動画・寝る」の3択から本当の生活感に化ける。**1モードずつ選択可**（例:「B2の読書だけ」）。

### B3. ライブ連動きりたん（音楽・タイマー・天気・時刻で行動が変わる） 【M】

- **土台**: Companionは天気・Spotify再生状態・ポモドーロ状態を既に持っている（`/api/state`）。壁紙は現在 `/api/ui`（設定）しか読んでいないため、**runtime情報を読むだけで連動できる**。DirectorのFSMにはdaypartバイアス機構が既にあり、外部バイアスの注入口を1つ足せばよい。
- **やること（v1、既存3+B1の4モードで成立する範囲）**:
  - Spotify再生中 → video_relax への遷移weight↑（「音楽かけてまったりモード」）
  - ポモドーロfocus中 → work系に固定・離席抑制、break中 → ストレッチ/離席weight↑
  - 深夜 → sleepy/sleep バイアス強化（既存daypart係数の活用）
  - B2採用後のv2: 雨→窓ガゼ、music_listen連動、おやつ時間帯→snack_break など
- **楽しさ**: 「自分がポモドーロ回してる間はきりたんも作業してる」「音楽をかけると寛ぎだす」— 壁紙が自分の行動に呼応する。

### B4. きりたんのひとりごと（吹き出し・オフライン完結） 【M】

- **土台**: モード遷移・daypart・天気・タイマー状態はすべて取得可能。吹き出しUIだけ新規。
- **やること**: 状況タグ（mode×daypart×天気×イベント）→セリフ候補の静的テーブル（JSON、後から増やせる）を作り、モード遷移時や時報に確率で1行を数秒表示。頭部ボーンのスクリーン座標近くに小さな吹き出し（重要: 頻度は控えめ・OFF設定つき）。
- **楽しさ**: 「そろそろ休憩すか」「雨降ってきたね」— APIキー不要・完全ローカルで人格が出る。**B5（AIチャット）の表示基盤の先行実装も兼ねる**。

### B5. AIチャット配線（休眠コードの起動） 【M〜L】

- **土台**: `chat_openai` / `chat_gemini`、`AiConfig`（人格プロンプト既定「あなたは東北きりたん。…」）、会話履歴の永続化まで**Rust側は実装済み**。無いのは `/api/ai/chat` ルートとUIだけ。
- **やること**: ①ルート追加（token必須・履歴上限）→ ②CompanionのCONTENTにチャットサブタブ → ③（任意）応答をB4の吹き出しに流して壁紙上で喋らせる。SYSTEM>APIキーのUIは既存のsecrets機構を利用。
- **楽しさ**: 常駐きりたんに話しかけられる。ニュースやタイマー状態をcontextに渡せば「今日のニュース何かあった?」まで行ける。
- **注意**: APIキー運用（コスト）と、キー無しでも成立するB4との住み分けを決めてから。

### B6. VOICEVOXきりたん読み上げ（個人ニュースを声で聞く） 【L・テーマ的本命】

- **土台**: 個人ニュースは行単位の進行制御（play/pause/行送り/章）とタイミング機構が完成済み。VOICEVOX（無料・ローカル）には東北きりたん音源が収録されており、HTTP API（localhost:50021）で合成できる。
- **やること**: ①Companionに「VOICEVOX連携」設定（接続確認・話者選択）→ ②再生時に行テキストを合成してCompanion側で再生 → ③行送りを音声終了と同期（現在の文字数比例タイマーの置き換えモード）。VOICEVOX未インストール時は従来動作。
- **楽しさ**: **きりたんが自分のニュース原稿を本人の声で読んでくれる**。このプロジェクトの題材的に一番「らしい」機能。
- **注意**: VOICEVOX別途インストールが前提。キャラ利用規約の範囲（個人利用）で。

### B7. TODOパネル（休眠モデルの起動） 【M】

- **土台**: `TodoItem`（priority/due_at付き）と永続化は実装済み。ルートとUIが無いだけ。
- **やること**: `/api/todos` CRUD → CompanionのCONTENTにTODOサブタブ → overlayにTODOパネル（メモパネルの実装パターンを流用、期限近いものを強調）。
- **楽しさ/実用**: 壁紙に「今日やること」が常時見える。ポモドーロと並ぶ実用枠。

### B8. 天気エフェクト（雨・雪を画面に降らせる） 【M】

- **土台**: Companionの天気は weatherCode 込みで取得済み。背景レイヤ（`SceneBackgroundLayer`）はCSSレイヤ構成で拡張しやすい。
- **やること**: weatherCode→エフェクト種別（雨/雪/なし）のマップ＋軽量パーティクルレイヤ（Canvas 2D、数百粒・FPS上限連動、OFF設定つき）。
- **楽しさ**: 現実で雨が降ると壁紙の部屋の窓の外も雨になる。B3と組むと「雨の日は外に出ない」等の行動連動も。

### B9. 時報・記念日イベント 【S〜M】

- **土台**: Directorへの時刻バイアスは実装済み。日付判定を足すだけ。
- **やること**: 正時にストレッチ/時計ガゼの確率アップ（時報）、日付テーブル（正月・クリスマス・**きりたん誕生日10/1** など）でB4のセリフや背景キュー・プリセットを切替。
- **楽しさ**: 「気づいたら誕生日仕様になってた」systemは長く使う壁紙ほど効く。

### B10. カーソル追視（オプション設定） 【S〜M・要相談】

- **土台**: `gazeController` は多層合成器として完成しており、マウスレイヤを1枚足すのは容易。
- **注意**: **マウス追尾は0.2で意図的に廃止した経緯がある**（壁紙として落ち着く待機ワンダー優先）。入れるなら「たまにチラ見する」程度の弱い追従＋既定OFFのCompanion設定を提案。WE側でもマウスイベント購読が必要。
- **楽しさ**: デスクトップを触っていると、たまに視線が付いてくる。

### B11. 昼夜の手動オーバーライド設定 【S】

- **土台**: `daypart.ts` の `resolveDaypart(override, date)` が**最初からoverride引数を持って待っている**（コメントにも「将来のCompanion設定用」と明記）。

### B12. ポストカード機能（今の一枚を保存） 【M】

- **土台**: Lab（`?lab=1`）にfreeze+capture経路が実装済み。製品経路が無いだけ。
- **やること**: Companion REMOTEに📷ボタン → `/api/ui` に screenshotSeq（タイマーのcommandSeq方式を流用）→ 壁紙がレンダー直後のフレームをPNG化しCompanionへPOST → データフォルダ`/screenshots/`に保存＋「フォルダを開く」。

### B13. オーディオビジュアライザ（WE専用） 【M〜L】
